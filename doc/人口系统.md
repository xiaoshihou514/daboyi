# 人口系统

## 概述

人口系统是大博弈经济仿真的核心之一。每个省份拥有若干人口群体（Pop），各群体按阶级划分，有不同的消费需求。人口的增长或衰减取决于需求满足程度，形成了对生产系统的动态压力。

## 人口阶级

游戏定义了 10 种人口阶级（`PopClass`），每种阶级在经济体系中扮演不同角色：

| 阶级 | 英文标识 | 经济角色 |
|------|----------|----------|
| 佃农 | TenantFarmer | 农场劳动力 |
| 自耕农 | Yeoman | 自耕农场劳动力 |
| 地主 | Landlord | 土地所有者 |
| 资本家 | Capitalist | 工业资本投入 |
| 小资产阶级 | PetitBourgeois | 手工业劳动力 |
| 宗教贵族 | Clergy | 宗教机构 |
| 官僚 | Bureaucrat | 行政机构 |
| 世俗贵族 | Nobility | 统治阶层 |
| 军队 | Soldier | 军事力量 |
| 知识分子 | Intelligentsia | 科技与文化 |

## 数据结构

```rust
// shared/src/lib.rs

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct Pop {
    pub class: PopClass,     // 阶级类型
    pub size: u32,           // 人口数量
    pub needs_satisfaction: f32,  // 需求满足度 (0.0–1.0)
}
```

每个省份的 `Province.pops` 字段是 `Vec<Pop>`，包含该省份所有人口群体。

## 消费需求

每种阶级每人每 tick（一天）消耗的商品量在 `server/src/game/data.rs` 的 `pop_needs()` 函数中定义：

| 阶级 | 粮食 | 衣物 | 燃料 | 工具 | 奢侈品 |
|------|------|------|------|------|--------|
| 佃农 | 0.002 | 0.0005 | 0.0003 | — | — |
| 自耕农 | 0.002 | 0.0005 | 0.0003 | 0.0001 | — |
| 地主 | 0.002 | 0.001 | 0.0005 | 0.0001 | 0.001 |
| 资本家 | 0.002 | 0.001 | 0.0005 | 0.0002 | 0.002 |
| 小资产阶级 | 0.002 | 0.0008 | 0.0004 | 0.0002 | 0.0005 |
| 宗教贵族 | 0.002 | 0.0008 | 0.0004 | 0.0001 | 0.001 |
| 官僚 | 0.002 | 0.0008 | 0.0004 | 0.0001 | 0.0008 |
| 世俗贵族 | 0.002 | 0.001 | 0.0005 | 0.0001 | 0.003 |
| 军队 | 0.003 | 0.001 | 0.0005 | 0.0003 | — |
| 知识分子 | 0.002 | 0.0008 | 0.0004 | 0.0002 | 0.001 |

**设计要点**：
- 所有阶级都消耗粮食，这是最基本的生存需求。
- 上层阶级（地主、资本家、贵族）有更高的奢侈品需求。
- 军队的粮食消耗高于平民（0.003 vs 0.002），反映军事后勤压力。

## 消费机制

消费逻辑在 `server/src/game/population.rs` 的 `consume()` 函数中实现：

```
每个 tick，对每个省份的每个 Pop 群体：
1. 计算总需求 = 每人需求 × 人口数量
2. 对于每种商品需求：
   a. 如果库存足够：扣减库存，该商品满足度 = 1.0
   b. 如果库存不足：耗尽库存，该商品满足度 = 库存 / 需求
3. 该 Pop 的 needs_satisfaction = 所有商品满足度的平均值
```

**关键特性**：
- 消费按 Pop 群体顺序执行——先消费的群体可能耗尽库存，导致后续群体挨饿。
- `needs_satisfaction` 是一个 0.0 到 1.0 的数值，直接驱动人口增长/衰减。

## 人口增长模型

增长逻辑在 `population.rs` 的 `grow()` 函数中实现，采用基于满足度的指数增长模型：

```
对于每个 Pop 群体：
├── 满足度 > 0.5（富足）：
│   delta = size × 0.001 × (satisfaction - 0.5) × 2
│   人口按每 tick 最高 0.1% 增长
├── 满足度 < 0.3（匮乏）：
│   delta = -size × 0.002 × (0.3 - satisfaction) / 0.3
│   人口按每 tick 最高 0.2% 衰减
└── 0.3 ≤ 满足度 ≤ 0.5（稳定区间）：
    无增减
```

**设计要点**：
- **稳定区间**（0.3–0.5）避免人口在临界点反复震荡。
- **不对称速率**：衰减速度（0.2%/tick）是增长速度（0.1%/tick）的两倍，饥荒比繁荣的影响更快更剧烈。
- 增长/衰减后人口最少为 1（不会完全灭绝）。

## 世界生成中的人口初始化

在 `server/src/game/data.rs` 的 `generate_world()` 中，每个省份的初始人口基于地图顶点数量启发式计算：

```
base_pop = max(vertices.len() × 2, 100)
```

顶点数量近似反映省份面积——面积越大的省份分配更多人口。初始人口按以下比例分配到各阶级：

| 阶级 | 比例 |
|------|------|
| 佃农 | 40% |
| 自耕农 | 25% |
| 小资产阶级 | 10% |
| 地主 | 5% |
| 官僚 | 5% |
| 宗教贵族 | 5% |
| 军队 | 5% |
| 知识分子 | 3% |
| 贵族 | 1% |
| 资本家 | 1% |

## 与其他系统的关联

```
生产系统 ──(产出商品)──→ 省份库存 ──(消费商品)──→ 人口需求满足
                                                    │
                                                    ↓
                                              需求满足度
                                                    │
                                          ┌─────────┴─────────┐
                                          ↓                   ↓
                                       人口增长             人口衰减
                                          │                   │
                                          ↓                   ↓
                                     更多可用劳动力      更少可用劳动力
                                          │                   │
                                          ↓                   ↓
                                     生产力提升           生产力下降
                                          └─────────┬─────────┘
                                                    ↓
                                              经济反馈循环
```

人口系统与生产系统形成了核心的经济反馈循环：
- 生产创造商品 → 满足人口需求 → 人口增长 → 更多劳动力 → 更高产出
- 生产不足 → 需求未满足 → 人口衰减 → 劳动力短缺 → 产出更低
