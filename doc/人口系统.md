# 人口系统

## 概述

人口系统是大博弈经济仿真的核心之一。每个省份拥有若干人口群体（Pop），各群体按阶级划分，有不同的消费需求。人口的增长或衰减取决于需求满足程度，形成了对生产系统的动态压力。

## 人口阶级

游戏定义了 10 种人口阶级（`PopClass`），每种阶级在经济体系中扮演不同角色：

| 阶级 | 英文标识 | 经济角色 |
|------|----------|----------|
| 佃农 | TenantFarmer | 农场劳动力 |
| 自耕农 | Yeoman | 自耕农场劳动力 |
| 地主 | Landlord | 土地所有者 |
| 资本家 | Capitalist | 工业资本投入 |
| 小资产阶级 | PetitBourgeois | 手工业劳动力 |
| 宗教贵族 | Clergy | 宗教机构 |
| 官僚 | Bureaucrat | 行政机构 |
| 世俗贵族 | Nobility | 统治阶层 |
| 军队 | Soldier | 军事力量 |
| 知识分子 | Intelligentsia | 科技与文化 |

## 数据结构

```rust
// shared/src/lib.rs

pub struct Pop {
    pub class: PopClass,
    pub size: u32,
    pub needs_satisfaction: f32,  // 0.0–1.0
}
```

每个省份的 `Province.pops` 字段是 `Vec<Pop>`，包含该省份所有人口群体。

## 消费机制

消费逻辑在 `server/src/game/population.rs` 的 `consume()` 函数中实现：

```
每个 tick，对每个省份的每个 Pop 群体：
1. 计算总需求 = 每人需求 × 人口数量
2. 对于每种商品需求：
   a. 如果库存足够：扣减库存，该商品满足度 = 1.0
   b. 如果库存不足：耗尽库存，该商品满足度 = 库存 / 需求
3. 该 Pop 的 needs_satisfaction = 所有商品满足度的平均值
```

## 人口增长模型

增长逻辑在 `population.rs` 的 `grow()` 函数中实现，采用基于满足度的指数增长模型：

```
对于每个 Pop 群体：
├── 满足度 > 0.5（富足）：
│   delta = size × 0.001 × (satisfaction - 0.5) × 2
│   人口按每 tick 最高 0.1% 增长
├── 满足度 < 0.3（匮乏）：
│   delta = -size × 0.002 × (0.3 - satisfaction) / 0.3
│   人口按每 tick 最高 0.2% 衰减
└── 0.3 ≤ 满足度 ≤ 0.5（稳定区间）：
    无增减
```

## 世界生成中的人口初始化

在 `server/src/game/data.rs` 的 `generate_world()` 中，人口初始化使用来自 EU5toGIS 项目的真实省份人口数据。

### 数据来源

- **EU5toGIS `06_pops_totals.txt`**：EU5 设计的各省份人口总量，复制到 `assets/pops.tsv`
- **格式**：每行 `province_tag\tpop_count`，约 2 万条记录
- **回退策略**：pops.tsv 未收录的省份，按地形密度 × 省份面积估算（粗略回退）

### 中世纪阶级分配比例

加载省份人口总量后，按以下固定比例分配到 10 种人口阶级：

| 阶级 | 比例 | 说明 |
|------|------|------|
| 佃农 | 50% | 最大群体，无地农民 |
| 自耕农 | 25% | 自有土地的农民 |
| 小资产阶级 | 6% | 手工业者、商贩 |
| 军队 | 4% | 职业军人、民兵 |
| 地主 | 4% | 拥有土地的地主 |
| 宗教贵族 | 4% | 教士、僧侣 |
| 官僚 | 3% | 行政人员 |
| 知识分子 | 2% | 学者、文人 |
| 世俗贵族 | 1.5% | 领主、公爵 |
| 资本家 | 0.5% | 1356 年极少 |

每个阶级的人口最少为 1 人（防止为 0）。

## 国家系统

### 来源

国家与省份归属**完全来自 EU5 存档**（`ti.eu5`），通过 `parse_save` 工具提取后存入 `assets/ownership.tsv`。服务端启动时通过 `load_eu5_ownership()` 加载。

### 国家创建流程

```
ownership.tsv（province_tag → owner_tag）
    └──→ 遍历所有可玩省份
          ├── 有主：记录 owner tag，取第一个已知省份作为首都
          └── 无主：province.owner = None（地图上显示为中性灰）
    └──→ 按 owner tag 去重 → 创建 Country 列表
    └──→ 从 country_names.tsv 查找中文名（qwen 翻译自 1356 年历史语境）
```

### Country 数据结构

```rust
pub struct Country {
    pub tag: String,           // 3 字母标签，如 "YUA"、"SWE"
    pub name: String,          // 中文显示名称（来自 country_names.tsv）
    pub capital_province: u32, // 首都省份 ID（该国在 ownership.tsv 中第一个出现的省份）
}
```

`GameState.countries` 存储所有活跃国家，`Province.owner` 存储所属国家标签字符串。

### 名称翻译

- **`assets/country_names.tsv`**：1,520 个国家标签 → 中文名，使用 qwen CLI 批量翻译，提示语境为"1356 年国家名称"
- **`assets/province_names.tsv`**：13,589 个省份标签 → 中文名，同样通过 qwen 翻译
- 翻译未覆盖的标签回退为原始标签字符串



## 与其他系统的关联

```
生产系统 ──(产出商品)──→ 省份库存 ──(消费商品)──→ 人口需求满足
                                                    │
                                                    ↓
                                              需求满足度
                                                    │
                                          ┌─────────┴─────────┐
                                          ↓                   ↓
                                       人口增长             人口衰减
                                          │                   │
                                          ↓                   ↓
                                     更多可用劳动力      更少可用劳动力
                                          │                   │
                                          ↓                   ↓
                                     生产力提升           生产力下降
                                          └─────────┬─────────┘
                                                    ↓
                                              经济反馈循环
```
