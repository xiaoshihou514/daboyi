# 人口系统

## 概述

人口系统是大博弈经济仿真的核心之一。每个省份拥有若干人口群体（Pop），各群体按阶级划分，有不同的消费需求。人口的增长或衰减取决于需求满足程度，形成了对生产系统的动态压力。

## 人口阶级

游戏定义了 10 种人口阶级（`PopClass`），每种阶级在经济体系中扮演不同角色：

| 阶级 | 英文标识 | 经济角色 |
|------|----------|----------|
| 佃农 | TenantFarmer | 农场劳动力 |
| 自耕农 | Yeoman | 自耕农场劳动力 |
| 地主 | Landlord | 土地所有者 |
| 资本家 | Capitalist | 工业资本投入 |
| 小资产阶级 | PetitBourgeois | 手工业劳动力 |
| 宗教贵族 | Clergy | 宗教机构 |
| 官僚 | Bureaucrat | 行政机构 |
| 世俗贵族 | Nobility | 统治阶层 |
| 军队 | Soldier | 军事力量 |
| 知识分子 | Intelligentsia | 科技与文化 |

## 数据结构

```rust
// shared/src/lib.rs

pub struct Pop {
    pub class: PopClass,
    pub size: u32,
    pub needs_satisfaction: f32,  // 0.0–1.0
}
```

每个省份的 `Province.pops` 字段是 `Vec<Pop>`，包含该省份所有人口群体。

| 自耕农 | 0.002 | 0.0005 | 0.0003 | 0.0001 | — |

## 消费机制

消费逻辑在 `server/src/game/population.rs` 的 `consume()` 函数中实现：

```
每个 tick，对每个省份的每个 Pop 群体：
1. 计算总需求 = 每人需求 × 人口数量
2. 对于每种商品需求：
   a. 如果库存足够：扣减库存，该商品满足度 = 1.0
   b. 如果库存不足：耗尽库存，该商品满足度 = 库存 / 需求
3. 该 Pop 的 needs_satisfaction = 所有商品满足度的平均值
```

## 人口增长模型

增长逻辑在 `population.rs` 的 `grow()` 函数中实现，采用基于满足度的指数增长模型：

```
对于每个 Pop 群体：
├── 满足度 > 0.5（富足）：
│   delta = size × 0.001 × (satisfaction - 0.5) × 2
│   人口按每 tick 最高 0.1% 增长
├── 满足度 < 0.3（匮乏）：
│   delta = -size × 0.002 × (0.3 - satisfaction) / 0.3
│   人口按每 tick 最高 0.2% 衰减
└── 0.3 ≤ 满足度 ≤ 0.5（稳定区间）：
    无增减
```

## 世界生成中的人口初始化

在 `server/src/game/data.rs` 的 `generate_world()` 中，人口初始化使用真实历史数据。

### 数据来源

- **OWID/HYDE 3.3 数据集**：全球各国从公元前 10000 年至 2023 年的历史人口估计
- **时间点**：1356 年（游戏开始日期），通过 1300 年和 1400 年数据线性插值得出
  - 公式：`pop_1356 = pop_1300 + (pop_1400 - pop_1300) × 56 / 100`
- **覆盖范围**：199 个国家/地区，使用 ISO-3166 alpha-3 代码

### 主要国家 1356 年人口

| 国家 | 人口 | 说明 |
|------|------|------|
| 中国（含台湾） | ~7,500 万 | 元末战乱，人口下降 |
| 印度 | ~1.26 亿 | 德里苏丹国时期 |
| 法国 | ~1,300 万 | 黑死病后恢复期 |
| 日本 | ~840 万 | 南北朝时期 |
| 德国 | ~760 万 | 神圣罗马帝国 |
| 英国 | ~333 万 | 黑死病后人口锐减 |

世界总人口约 **5.08 亿**。

### 人口分配流程

```
1. 按国家代码聚合省份总面积
2. 查找国家 1356 年总人口（OWID 数据）
3. 按面积比例分配到各省份
4. 按中世纪阶级比例分配到各人口群体
```

### 中世纪阶级分配比例

| 阶级 | 比例 | 说明 |
|------|------|------|
| 佃农 | 50% | 最大群体，无地农民 |
| 自耕农 | 25% | 自有土地的农民 |
| 小资产阶级 | 6% | 手工业者、商贩 |
| 军队 | 4% | 职业军人、民兵 |
| 地主 | 4% | 拥有土地的地主 |
| 宗教贵族 | 4% | 教士、僧侣 |
| 官僚 | 3% | 行政人员 |
| 知识分子 | 2% | 学者、文人 |
| 世俗贵族 | 1.5% | 领主、公爵 |
| 资本家 | 0.5% | 1356 年极少 |

### 面积计算

省份面积使用鞋带公式（Shoelace formula）近似计算，乘以 `cos(纬度)` 校正因子补偿高纬度经线收敛。

### 回退策略

OWID 数据未覆盖的国家/地区使用面积估计回退：每平方度约 60,000 人（约等于中世纪平均 5 人/km²）。

## 国家系统

### 历史国家分配

游戏在 1356 年开局时，根据历史疆域数据为每个省份分配所属国家。

#### 数据来源

- **全球基础数据**：`aourednik/historical-basemaps` 项目的 `world_1400.geojson`
  - 包含 115 个命名政体的多边形边界
  - 使用 1400 年数据作为 1356 年近似（相差 44 年）
- **中国地区**：按行政区划代码（GB 码前缀）硬编码分配至七大势力

#### 中国七雄（~1356 年）

| 标签 | 名称 | 控制区域（现代省份） | 历史背景 |
|------|------|---------------------|----------|
| YUA | 元朝 | 北京、天津、河北、山西、内蒙古、辽宁、吉林、黑龙江、山东、陕西、甘肃、青海、宁夏、新疆、西藏、福建、广东、广西、海南 | 蒙古帝国残余，控制北方及边疆 |
| HSG | 韩宋 | 河南 | 刘福通红巾军政权 |
| ZYZ | 朱元璋 | 安徽 | 1356 年刚攻占集庆（南京），后建明朝 |
| CYL | 陈友谅 | 湖北、湖南、江西 | 天完政权→汉政权 |
| ZSC | 张士诚 | 上海、江苏 | 控制苏州等富庶地区 |
| FGZ | 方国珍 | 浙江 | 东南沿海势力 |
| XIA | 明夏 | 重庆、四川、贵州、云南 | 明玉珍大夏政权 |

- 台湾（GB 码 1567xx）分配为 `UNC`（未开化）
- 蒙古（country_code = MNG）分配为 `YUA`（元朝）

#### 全球国家分配

对非中国省份，使用点-多边形（point-in-polygon）算法：
1. 加载 `world_1400.geojson` 中的历史政体边界
2. 跳过 "Great Khanate"（已由中国七雄替代）
3. 取每个省份的质心坐标
4. 用 `geo::Contains` 判断质心落入哪个历史政体
5. 匹配到的政体名通过 `country_tag_for_name()` 映射为 3 字母国家标签
6. 未匹配的省份标记为 `UNC`（未开化）

#### Country 数据结构

```rust
pub struct Country {
    pub tag: String,           // 3 字母标签，如 "YUA"、"FRA"
    pub name: String,          // 显示名称
    pub capital_province: u32, // 首都省份 ID
}
```

`GameState.countries` 存储所有国家，`Province.owner` 存储所属国家标签。

## 与其他系统的关联

```
生产系统 ──(产出商品)──→ 省份库存 ──(消费商品)──→ 人口需求满足
                                                    │
                                                    ↓
                                              需求满足度
                                                    │
                                          ┌─────────┴─────────┐
                                          ↓                   ↓
                                       人口增长             人口衰减
                                          │                   │
                                          ↓                   ↓
                                     更多可用劳动力      更少可用劳动力
                                          │                   │
                                          ↓                   ↓
                                     生产力提升           生产力下降
                                          └─────────┬─────────┘
                                                    ↓
                                              经济反馈循环
```
