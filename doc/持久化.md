# 持久化系统

## 概述

服务端使用 RocksDB 作为嵌入式键值数据库，实现游戏状态的断点续玩。持久化策略注重简洁性：以单一键存储完整状态快照，采用节流写入避免 I/O 瓶颈。

## 技术选型

| 项目 | 选择 | 理由 |
|------|------|------|
| 数据库 | RocksDB 0.24 | 嵌入式、高性能、无需外部进程 |
| 序列化 | JSON (serde_json) | 人类可读，便于调试和检查 |
| 存储模式 | 单键全量 | 实现简单，状态一致性有保障 |

## 数据库层实现

### 位置

`server/src/db.rs`

### 接口

```rust
pub struct GameDb {
    db: rocksdb::DB,
}

impl GameDb {
    /// 打开或创建数据库
    pub fn open(path: &str) -> Result<Self>;

    /// 保存完整游戏状态
    pub fn save_state(&self, state: &GameState) -> Result<()>;

    /// 加载游戏状态
    pub fn load_state(&self) -> Result<Option<GameState>>;
}
```

### 存储格式

- **键**：`"game_state"`（固定字符串）
- **值**：`serde_json::to_string(&state)` 的 JSON 字符串
- **数据库路径**：`./daboyi.db`（项目根目录下）

## 写入策略

### 节流写入

```rust
// server/src/ws.rs

if gs.tick % 300 == 0 {
    state.db.lock().unwrap().save_state(&gs).ok();
}
```

每 **300 个 tick** 写入一次，而非每 tick 都写。以默认 1 秒/tick 的速度，这意味着约每 5 分钟持久化一次。

**设计理由**：
- RocksDB 写入涉及 WAL（预写日志）和后台 compaction，频繁写入会增加 I/O 负担。
- 游戏状态序列化体积较大（bincode ~5MB），过于频繁的写入对磁盘压力过大。
- 300 tick 的间隔在数据安全与性能之间取得平衡——最多丢失约 300 天的游戏进度（约 10 个月）。

## 启动恢复流程

```rust
// server/src/main.rs

let db = GameDb::open("./daboyi.db")?;

let game_state = match db.load_state()? {
    Some(state) => {
        println!("Loaded saved game (tick {})", state.tick);
        state
    }
    None => {
        // 首次启动：从 map.bin 生成新世界
        match MapData::load("assets/map.bin") {
            Ok(map) => generate_world(&map),
            Err(_) => GameState::default(),  // 无地图则空状态
        }
    }
};
```

### 恢复优先级

```
1. RocksDB 中有保存的状态 → 恢复（断点续玩）
2. RocksDB 为空 + map.bin 存在 → 生成新世界
3. RocksDB 为空 + map.bin 缺失 → 空 GameState
```

## 数据一致性

### 当前保证

- **原子性**：单键写入是原子操作——要么完整写入，要么不写入。
- **一致性**：完整状态快照确保所有省份数据在同一时刻一致。
- **持久性**：RocksDB 的 WAL 保证已确认的写入不会因崩溃丢失。

### 局限

- **无增量备份**：每次写入完整状态，无法回退到特定时刻。
- **无版本管理**：覆盖写入，无历史状态保留。
- **JSON 体积**：约 37MB 的 JSON 文本，可考虑改用 bincode 减小体积。

## 未来优化方向

| 方向 | 说明 |
|------|------|
| bincode 持久化 | 将存储格式从 JSON 改为 bincode，减小 I/O 体积 |
| 增量保存 | 仅写入自上次保存以来变化的省份 |
| 多存档槽 | 支持多个保存槽位，键为 `"save_1"`、`"save_2"` 等 |
| 自动存档 | 按游戏内日期（如每月初）自动创建检查点 |
| 压缩 | 启用 RocksDB 的 zstd 压缩减小磁盘占用 |
