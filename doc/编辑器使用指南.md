# 编辑器使用指南

本文档介绍如何使用 Daboyi 地图编辑器进行替代历史地图着色：加载资产、绘制省份、管理国家与行政区，以及保存/加载着色文件。

---

## 启动编辑器

确保已完成资产准备（详见[数据提取指南](数据提取.md)）后，在仓库根目录运行：

```bash
cargo run -p client
```

编辑器窗口打开后，地图默认以**政治模式**显示。若 `coloring.json` 存在于运行目录，则自动加载上次的着色状态；否则使用 `assets/ownership.tsv` 和 `assets/country_colors.tsv` 作为默认状态。

## 地图模式

通过工具栏按钮或键盘快捷键切换：

| 快捷键 | 模式 | 用途 |
|--------|------|------|
| `1` | **省份模式** | 显示 EU5 原始省份识别颜色，方便对照参考地图 |
| `2` | **地形模式** | 高亮显示荒地与水域，方便识别不可用区域 |
| `3` | **政治模式** | 按国家/行政区着色，编辑时使用此模式（默认） |

## 管理国家

### 新建国家

在左侧面板点击**「+ 新建国家」**，输入国家标签（3 字母，如 `MNG`）和显示名称（如 `蒙古`）。新国家将以随机颜色添加至列表。

### 重命名国家

点击国家旁的**「重命名」**按钮，在弹出输入框中修改名称后确认。

### 删除国家

点击国家旁的**「删除」**按钮。删除后，该国家的所有省份指定将被清除，省份变为无主（显示灰色）。

### 修改国家颜色

点击国家名称旁的颜色块，在颜色选择器中调整 RGBA 值。地图上该国家的所有省份颜色实时更新。

### 设置首都

在右侧省份面板中，选中目标省份后点击**「设为首都」**，将当前国家的首都更新为该省份。首都位置以星形标记显示在地图上。

## 管理行政区

行政区（`AdminArea`）支持无限层级嵌套，形成 ADM1 → ADM2 → ADM3 … 的树形结构。行政区的颜色优先级高于国家颜色。

### 新建行政区

1. 在左侧面板选中某个国家（或某个行政区作为父级）
2. 点击**「+ 新建子区域」**
3. 输入区域名称（如 `漠北`、`岭北行省`）

新行政区默认继承父级颜色（`color: null`）。

### 为行政区指定颜色

点击行政区旁的颜色块，设置独立颜色。设置后，该行政区内的省份将以此颜色显示，不再继承国家颜色。若清空颜色（设为 null），则重新继承父级颜色。

### 颜色继承规则

```
国家颜色
  └─ ADM1 颜色（若为 null，继承国家颜色）
       └─ ADM2 颜色（若为 null，继承 ADM1 颜色）
            └─ ADM3 颜色（以此类推）
```

### 删除行政区

点击行政区旁的**「删除」**按钮。删除后：
- 该行政区内所有省份的行政区指定被清除
- 子行政区也将被级联删除

## 绘制省份

### 基本流程

1. 切换到**政治模式**（快捷键 `3`）
2. 在左侧面板**单击选中**目标国家或行政区（选中后高亮显示）
3. 在地图上**左键单击**或**按住左键拖拽**，将省份涂为选中的国家/行政区

### 涂色逻辑

- 若选中的是**国家**：省份被归属到该国家（更新 `assignments`）
- 若选中的是**行政区**：省份同时归属到该行政区及其所属国家（更新 `admin_assignments` 和 `assignments`），显示行政区颜色

### 撤销归属

目前编辑器不支持撤销（Ctrl+Z）。若需取消某个省份的归属：

1. 在左侧面板取消选中所有国家和行政区（点击已选中的项再次点击，或点击空白区域）
2. 左键点击目标省份，省份将变为无主（灰色）

## 保存与加载

### 保存

点击工具栏的**「保存」**按钮（或未来支持的快捷键），将当前着色状态写入运行目录下的 `coloring.json`。

**文件格式示例：**

```json
{
  "countries": [
    {
      "tag": "MNG",
      "name": "蒙古",
      "color": [0.8, 0.2, 0.1, 1.0],
      "capital_province": 1234
    }
  ],
  "assignments": {
    "1234": "MNG",
    "5678": "MNG"
  },
  "admin_areas": [
    {
      "id": 1,
      "name": "漠北",
      "country_tag": "MNG",
      "parent_id": null,
      "color": [0.9, 0.3, 0.2, 1.0]
    },
    {
      "id": 2,
      "name": "岭北行省",
      "country_tag": "MNG",
      "parent_id": 1,
      "color": null
    }
  ],
  "admin_assignments": {
    "9999": 1,
    "8888": 2
  }
}
```

### 加载

点击工具栏的**「加载」**按钮，从 `coloring.json` 读取着色状态并覆盖当前编辑内容。**此操作不可撤销**，加载前请确认当前未保存的更改已不再需要。

## 典型工作流示例

以下是绘制一幅元末中国地图的典型流程：

```
1. 运行 cargo run -p client 打开编辑器
2. 按 3 切换到政治模式
3. 在左侧面板点击「+ 新建国家」，添加：
   - MNG 蒙古（蓝色）
   - MII 明（红色）
   - ...
4. 点击「蒙古」，拖拽涂色蒙古控制区域
5. 为蒙古添加行政区：
   - 「+ 新建子区域」→ 岭北行省（浅蓝色）
   - 「+ 新建子区域」→ 辽阳行省（深蓝色）
6. 点击「岭北行省」，涂色对应省份
7. 点击工具栏「保存」，写入 coloring.json
```

## 常见问题

**Q：启动时地图为空白？**

检查 `assets/map.bin` 和 `assets/terrain.bin` 是否存在。若不存在，请运行 `cargo run --release -p mapgen` 生成。

**Q：省份没有中文名称？**

检查 `assets/province_names.tsv` 是否存在。若不存在，请运行 `python3 tools/extract_translations.py` 生成。

**Q：打开后地图全是灰色？**

灰色表示省份无主。若希望使用 EU5 默认归属作为起点，确保 `assets/ownership.tsv` 和 `assets/country_colors.tsv` 存在（通过 `cargo run -p parse_save` 生成），且 `coloring.json` 不存在或已删除。

**Q：coloring.json 保存在哪里？**

保存在程序运行目录（即仓库根目录）下，文件名为 `coloring.json`。
