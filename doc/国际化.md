# 国际化（i18n）系统

## 概述

大博弈的 UI 采用**中文优先**策略：所有界面文本默认显示简体中文，并通过 `rust-i18n` 库支持多语言扩展。省份与国家名称由 AI（qwen CLI）批量翻译，结合 1356 年历史语境。

## 技术实现

### rust-i18n

- **版本**：rust-i18n 3
- **语言文件**：`locales/zh.yml`（主语言）、`locales/en.yml`（备用）
- **初始化**（`client/src/main.rs`）：

```rust
rust_i18n::i18n!("../locales", fallback = "zh");
// 在 main() 中：
rust_i18n::set_locale("zh");
```

- **使用**：所有 UI 字符串通过 `t!("key")` 宏访问

```rust
Text::new(t!("connecting").to_string())
TextFont { font: cjk.clone(), font_size: 18.0, ..default() }
```

### 语言文件格式

rust-i18n v3 使用**平坦键**格式（文件名即语言代码）：

```yaml
# locales/zh.yml
_version: 1
connecting: "连接中..."
click_province: "点击省份"
paused: "暂停"
owner: "归属"
# ...
```

> **注意**：v3 不使用顶层语言键（如 `zh: { key: value }`），文件名本身就是语言标识。

### 添加新 UI 字符串

1. 在 `locales/zh.yml` 添加键值对（中文）
2. 在 `locales/en.yml` 添加对应英文
3. 代码中使用 `t!("new_key")`

## CJK 字体

Bevy 默认字体不含 CJK 字形，需单独加载中文字体。

### 字体文件

- **文件**：`assets/fonts/NotoSansCJKsc-Regular.otf`（16 MB）
- **来源**：从系统 NotoSansCJK TTC 字体集合中提取（Simplified Chinese 字体）

### 加载方式

字体在 `UiPlugin::setup()` 启动时加载，存储为 Bevy 全局资源：

```rust
#[derive(Resource)]
pub struct CjkFont(pub Handle<Font>);

fn setup(mut commands: Commands, asset_server: Res<AssetServer>) {
    let cjk: Handle<Font> = asset_server.load("fonts/NotoSansCJKsc-Regular.otf");
    commands.insert_resource(CjkFont(cjk.clone()));
    // 所有 TextFont 组件都使用 font: cjk.clone()
}
```

### Bevy AssetPlugin 配置

字体和其他资源统一从工作区 `assets/` 目录加载（而非 `client/assets/`），通过 `AssetPlugin` 配置：

```rust
.set(AssetPlugin {
    file_path: concat!(env!("CARGO_MANIFEST_DIR"), "/../assets").to_string(),
    ..default()
})
```

## 省份与国家名称翻译

### 来源文件

| 文件 | 条目数 | 说明 |
|------|--------|------|
| `assets/province_names.tsv` | 13,589 | 省份标签 → 中文名 |
| `assets/country_names.tsv` | 1,520 | 国家标签 → 中文名 |

格式为 `tag\tchinese_name`，带表头行。

### 翻译流程

使用 `qwen` CLI（`/usr/local/bin/qwen`）批量翻译：

**省份名称**（批次大小 500）：
```
提示语：你在翻译1356年的省份名称，请翻译以下省份名，每行一条，输出格式「英文名\t中文名」，不要输出其他内容。
输入：省份标签转为标题格式（underscores → spaces, title case）
```

**国家标签**（批次大小 100）：
```
提示语：你在翻译1356年的国家名称。以下每行是「3字母标签」，标签本身即对应该国英文名缩写，请翻译成1356年时期准确的中文历史名称，输出格式「标签\t中文名」，每行一条，不要输出其他内容。
输入：三字母国家标签（qwen 根据标签推断英文名）
```

翻译覆盖率约 99%，未翻译的标签回退为原始标签字符串。

### 服务端加载

```rust
// server/src/game/data.rs
fn load_province_names() -> HashMap<String, String>  // assets/province_names.tsv
fn load_country_names() -> HashMap<String, String>   // assets/country_names.tsv
```

在 `generate_world()` 中，这两个映射用于设置 `Province.name` 和 `Country.name`。

### 重新生成翻译

若需重新翻译，可使用以下脚本模板：

```python
import subprocess, re

tags = ["AAC", "CHI", ...]  # 读取 country_names.tsv 第一列
for i in range(0, len(tags), 100):
    batch = "\n".join(tags[i:i+100])
    r = subprocess.run(["qwen", "-p", "你在翻译1356年的国家名称..."],
                       input=batch, capture_output=True, text=True)
    for line in r.stdout.splitlines():
        m = re.match(r'^([A-Z]{2,5})\t(.+)$', line)
        if m:
            results[m.group(1)] = m.group(2)
```

## 新增 UI 功能时的约定

1. **只用 `t!()` 宏**，不硬编码中文字符串
2. **所有 `TextFont` 必须指定** `font: cjk.clone()`（否则中文字形无法显示）
3. 先在 `locales/zh.yml` 中加中文，再在 `locales/en.yml` 中加英文
4. 商品、人口阶级、建筑类型的显示名分别通过 `good_name()`、`pop_class_name()`、`building_name()` 辅助函数（`client/src/ui/mod.rs`）获取
