# 架构总览

## 项目简介

大博弈（daboyi）是一款以 Rust 编写的大战略游戏，开局时间为 **1356 年**（元末群雄割据）。采用客户端-服务器架构，服务端负责游戏逻辑仿真与持久化，客户端负责地图渲染与用户交互，二者通过 WebSocket 通信。

## 技术栈

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| 服务端框架 | actix-web 4 | 异步 HTTP/WebSocket 服务器 |
| 持久化 | RocksDB 0.24 | 嵌入式键值数据库 |
| 客户端引擎 | Bevy 0.15 | Rust 原生 ECS 游戏引擎 |
| 共享类型 | shared crate | 客户端与服务端共用的数据结构 |
| 地图编译 | mapgen 工具 | GeoPackage (GPKG) → bincode 离线处理 |
| 序列化 | serde + bincode/JSON | 命令用 JSON，状态快照用 bincode |
| 国际化 | rust-i18n 3 | 中文优先 UI，YAML 语言文件 |
| 字体 | NotoSansCJKsc-Regular.otf | 简体中文渲染 |
| 编译工具链 | clang + mold | clang 编译 RocksDB，mold 加速链接 |

## 核心设计原则

### 1. 服务端权威架构

所有游戏逻辑在服务端执行。客户端是无状态的——每个 tick 接收完整的 `GameState` 快照后重新渲染。

### 2. 客户端驱动的 tick 模型

客户端按 `GameSpeed` 计时器的频率向服务端发送 `Tick` 消息。服务端收到后推进一天的仿真，将完整状态返回。玩家可按空格键暂停/恢复。

### 3. 非对称序列化

- **客户端 → 服务端**：JSON 文本帧（可读性优先）
- **服务端 → 客户端**：bincode 二进制帧（~5MB vs JSON ~37MB）

### 4. 经济节流

人口消费、生产等经济计算每 **100 tick**（约 100 天）执行一次，日期每 tick 推进一天。存档每 **300 tick** 写入 RocksDB。

### 5. 安全数值转换

所有 `as` 类型转换集中在 `shared/src/conv.rs` 中。业务逻辑通过命名函数（如 `f64_to_u32`、`usize_to_f32`）调用，避免隐式截断和溢出。无 `unsafe` 代码块。

## 数据流

```
离线准备：
  EU5toGIS datasets/
    locations.gpkg ──→ mapgen ──→ assets/map.bin（省份几何）
    ports.gpkg     ──┘         └→ assets/terrain.bin（地形背景）
    rivers.tif                 → assets/rivers.png（河流叠加）
    06_pops_totals.txt ──────────→ assets/pops.tsv（人口）
  ti.eu5 ──→ parse_save ─────────→ assets/ownership.tsv（所有权）
  qwen CLI ──────────────────────→ assets/province_names.tsv（省份中文名）
                                  assets/country_names.tsv（国家中文名）

启动时（服务端）：
  map.bin + ownership.tsv + pops.tsv + *_names.tsv
    └──→ generate_world()
          ├── 人口：从 pops.tsv 读取真实省份人口数据
          ├── 所有权：从 ownership.tsv 读取 EU5 存档省份归属
          ├── 国家：从有主省份自动推导国家列表
          ├── 名称：province_names.tsv 与 country_names.tsv
          └── 建筑：按人口规模初始化农场与木炭窑
    └──→ 持久化至 RocksDB（后续启动直接加载）

启动时（客户端）：
  map.bin ──→ 构建合并省份网格（单 Mesh2d 实体）
  terrain.bin ──→ 构建合并地形网格（单 Mesh2d 实体）
  rivers.png ──→ 精灵叠加（NearestNeighbor 采样，flip_y）
  fonts/NotoSansCJKsc-Regular.otf ──→ CjkFont 资源（所有 TextFont 使用）

运行时（每 tick）：
  客户端 ──(Tick JSON)──→ 服务端
  服务端: advance() → 日期推进 → [每100tick: 生产→消费→增长]
  服务端 ──(StateSnapshot bincode)──→ 客户端
  客户端: 更新顶点颜色 → 更新 HUD
```

## 数据流

```
1. 离线编译：
   EU5toGIS GPKG 数据 ──→ mapgen（读取+坐标变换+简化+三角剖分）──→ assets/map.bin

2. 启动时：
   服务端加载 map.bin + world_1400.geojson ──→ generate_world()
     ├── 历史国家分配（point-in-polygon + 中国七雄，基于地理坐标）
     ├── 地形人口估算（基于 topography/vegetation/climate，总量缩放至 5 亿）
     ├── 资源映射（EU5 raw_material 51 种 → 7 种游戏商品）
     └── 建筑初始化
   客户端加载 map.bin ──→ 构建合并网格

3. 运行时（每 tick）：
   客户端 ──(Tick JSON)──→ 服务端
   服务端: advance() → 日期推进 → [每100tick: 生产→消费→增长]
   服务端 ──(StateSnapshot bincode)──→ 客户端
   客户端: 更新顶点颜色 → 更新 HUD

4. 持久化：
   服务端每 300 tick 将 GameState 序列化存入 RocksDB
```
