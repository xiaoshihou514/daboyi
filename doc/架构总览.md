# 架构总览（地图编辑器）

## 项目简介

Daboyi 是一款以 Rust 和 Bevy 构建的**替代历史地图编辑器**。用户可在 EU5 省份地图上绘制自定义国家与行政区划，并将着色结果保存为 JSON 文件。项目为单进程架构，无服务端、无网络通信，编辑器直接作为 Bevy 桌面应用运行。

---

## 技术栈

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| 引擎 | Bevy 0.15 | Rust 原生 ECS 游戏引擎，2D 渲染 |
| 语言 | Rust stable（2021 edition） | |
| 编译工具链 | clang + mold | mold 加速链接，clang 编译 C 依赖 |
| 序列化 | serde + JSON / bincode | 着色文件用 JSON，地图几何用 bincode |
| 字体 | NotoSansCJKsc-Regular.otf | 简体中文渲染 |
| 地图数据 | EU5toGIS GeoPackage | 约 22,688 个全球省份多边形 |

## 插件架构

编辑器由若干 Bevy 插件组成，职责明确：

```
Bevy App
├── MapPlugin       — 省份网格渲染（单一合并 Mesh2d）、着色模式切换
│   └── BordersPlugin — 相邻省份边界线渲染（所有者/行政区不同时显示）
├── TerrainPlugin   — 荒地与水域背景网格（单一合并 Mesh2d）
├── CapitalsPlugin  — 首都星形标记（按国家 capital_province 定位）
└── UiPlugin        — egui/Bevy UI 界面
    ├── 左侧面板：国家与行政区树形列表（新建/重命名/删除/选中）
    ├── 右侧面板：当前选中省份的详细信息
    └── 工具栏：地图模式按钮（省份/地形/政治）、保存/加载按钮
```

## 核心资源（Resources）

所有编辑器状态均存储为 Bevy `Resource`，在 `editor.rs` 中定义：

| 资源 | 类型 | 说明 |
|------|------|------|
| `MapColoring` | `HashMap<u32, String>` | 省份 ID → 国家标签 |
| `EditorCountries` | `Vec<EditorCountry>` | 所有国家列表 |
| `AdminAreas` | `Vec<AdminArea>` | 所有行政区列表（支持无限层级） |
| `AdminAssignments` | `HashMap<u32, u32>` | 省份 ID → 行政区 ID |
| `ActiveCountry` | `Option<String>` | 当前选中国家的标签 |
| `ActiveArea` | `Option<u32>` | 当前选中行政区的 ID |
| `NextAreaId` | `u32` | 行政区 ID 自增计数器 |
| `ProvinceNames` | `HashMap<u32, String>` | 省份 ID → 中文名称 |
| `MapMode` | 枚举 | 当前地图显示模式 |

## 核心数据类型

定义于 `shared/src/lib.rs`：

```rust
pub struct EditorCountry {
    pub tag: String,               // 3 字母标签，如 "MNG"
    pub name: String,              // 中文显示名称
    pub color: [f32; 4],           // RGBA 颜色
    pub capital_province: u32,     // 首都省份 ID
}

pub struct AdminArea {
    pub id: u32,
    pub name: String,
    pub country_tag: String,
    pub parent_id: Option<u32>,    // None = 顶级行政区（ADM1）
    pub color: Option<[f32; 4]>,   // None = 继承父级或国家颜色
}

pub struct ColoringFile {
    pub countries: Vec<EditorCountry>,
    pub assignments: HashMap<u32, String>,         // 省份 ID → 国家标签
    pub admin_areas: Vec<AdminArea>,
    pub admin_assignments: HashMap<u32, u32>,      // 省份 ID → 行政区 ID
}
```

## 省份着色优先级

地图渲染时，省份颜色按以下优先级确定：

1. **行政区指定**：`admin_assignments[id]` 存在 → 使用行政区颜色（或向上继承至国家）
2. **国家指定**：`assignments[id]` 存在 → 使用国家颜色
3. **无主省份**：显示中性灰（`[0.55, 0.55, 0.55, 1.0]`）

## 数据流

```
离线准备（一次性，克隆仓库后执行）：
  EU5toGIS datasets/
    locations.gpkg ──→ cargo run -p mapgen ──→ assets/map.bin
    ports.gpkg     ──┘                      └→ assets/terrain.bin
  .eu5 存档 ────────→ cargo run -p parse_save → assets/ownership.tsv
                                               assets/vassals.tsv
                                               assets/merchandize.tsv
                                               assets/country_colors.tsv

启动时（client）：
  map.bin ──────────→ 合并省份网格（单 Mesh2d，每省份一段顶点）
  terrain.bin ──────→ 合并地形网格（单 Mesh2d）
  ownership.tsv ────→ 初始 MapColoring 资源（历史归属作为默认值）
  country_colors.tsv→ 初始 EditorCountries 颜色
  province_names.tsv→ ProvinceNames 资源
  coloring.json ────→ （若存在）加载用户着色覆盖默认值

运行时（用户操作）：
  左键/拖拽 ────────→ interact.rs 更新 MapColoring / AdminAssignments
  着色变更 ─────────→ map/mod.rs 重新计算顶点颜色 → 更新 Mesh 顶点属性
  保存按钮 ─────────→ editor.rs::save_coloring() → 写入 coloring.json
  加载按钮 ─────────→ editor.rs::load_coloring() → 读取 coloring.json → 刷新资源
```

## 编译配置

`.cargo/config.toml`：

```toml
[build]
rustflags = ["-C", "link-arg=-fuse-ld=mold"]

[env]
CC = "clang"
CXX = "clang++"
```

- **mold**：超高速链接器，大幅缩短增量编译时间
- **clang**：编译部分 C/C++ 依赖库

---

## 编码约定

- **允许基础类型使用 `as` 转换**：`u32`/`usize`/`f32`/`f64` 等基础数值类型可直接转换；需要范围限制时在调用点显式处理
- **禁止 `unsafe` 代码**：代码库中无任何 `unsafe` 块
- **禁止 `rm` 删除文件**：使用 `kioclient move "file://..." 'trash:/'`
