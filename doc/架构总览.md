# 架构总览

## 项目简介

大博弈（daboyi）是一款以 Rust 编写的大战略游戏，开局时间为 **1356 年**（元末群雄割据）。采用客户端-服务器架构，服务端负责游戏逻辑仿真与持久化，客户端负责地图渲染与用户交互，二者通过 WebSocket 通信。

## 技术栈

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| 服务端框架 | actix-web 4 | 异步 HTTP/WebSocket 服务器 |
| 持久化 | RocksDB 0.24 | 嵌入式键值数据库 |
| 客户端引擎 | Bevy 0.15 | Rust 原生 ECS 游戏引擎 |
| 共享类型 | shared crate | 客户端与服务端共用的数据结构 |
| 地图编译 | mapgen 工具 | GeoPackage (GPKG) → bincode 离线处理 |
| 序列化 | serde + bincode/JSON | 命令用 JSON，状态快照用 bincode |
| 国际化 | rust-i18n 3 | 中文优先 UI，YAML 语言文件 |
| 字体 | NotoSansCJKsc-Regular.otf | 简体中文渲染 |
| 编译工具链 | clang + mold | clang 编译 RocksDB，mold 加速链接 |

## 游戏状态机

```
StartScreen ──(点击"开始游戏")──→ CountrySelection ──(点击"以此国开始")──→ Playing
     ↑                                    │
     └──────────(点击"返回")──────────────┘
```

| 状态 | 说明 |
|------|------|
| **StartScreen** | 全屏标题覆盖层，政治地图在背景渲染但不可交互 |
| **CountrySelection** | 政治地图；点击省份显示所属国家；顶/底半透明栏；可点击"以此国开始" |
| **Playing** | 正式游玩；HUD、省份面板、首都标记、地图模式切换、Tick 发送均激活 |

## 核心数据类型

### Country（国家）

```rust
pub struct Country {
    pub tag: String,                        // 3 字母标签，如 "SWE"、"YUA"
    pub name: String,                       // 中文显示名称（来自 country_names.tsv）
    pub capital_province: u32,              // 首都省份 ID（第一个已知省份）
    pub produced_goods: Vec<(String, f32)>, // 上月产出商品（EU5 原始名，按产量降序）
}
```

### GameState

```rust
pub struct GameState {
    pub date: GameDate,
    pub provinces: Vec<Province>,    // ~21,111 个可玩省份
    pub countries: Vec<Country>,
    pub vassals: HashMap<String, String>, // 附庸 → 宗主
    pub building_types: Vec<BuildingType>,
}
```

### 宗藩关系

`vassals`：存档中 `dependency` 块解析为 `subject → overlord` 映射，涵盖所有依附类型。政治地图中附庸国使用宗主国颜色，通过 `resolve_sovereign()` 最多追溯 10 跳防止循环。

### 政治地图着色

省份颜色优先级：

1. 荒地 → 地形色
2. 有主省份 → 宗主国 `country_colors.tsv` 中的 EU5 原始颜色
3. 无 TSV 条目 → FNV-1a 哈希兜底色
4. 无主省份 → 中性灰（`[0.55, 0.55, 0.55, 1.0]`）

## 核心设计原则

### 1. 服务端权威

所有游戏逻辑在服务端执行。客户端是无状态的——每个 tick 接收完整的 `GameState` 快照后重新渲染。

### 2. 客户端驱动的 tick 模型

客户端按 `GameSpeed` 计时器频率向服务端发送 `Tick` 消息。服务端推进一天的仿真后返回完整状态。空格键暂停/恢复。

### 3. 非对称序列化

- **客户端 → 服务端**：JSON 文本帧（可读性优先，< 1KB）
- **服务端 → 客户端**：bincode 二进制帧（~5 MB vs JSON ~37 MB）

### 4. 经济节流

经济计算（生产、人口消费与增长）每 **100 tick**（约 100 天）执行一次，日期每 tick 推进一天。存档每 **300 tick** 写入 RocksDB。

### 5. 无 unsafe、无 as 转换

所有数值类型转换集中在 `shared/src/conv.rs`，通过命名函数调用（`f64_to_u32`、`usize_to_f32` 等）。代码库中无 `unsafe` 块。

## 数据流

```
离线准备：
  EU5toGIS datasets/
    locations.gpkg ──→ mapgen ──→ assets/map.bin（省份几何）
    ports.gpkg     ──┘         └→ assets/terrain.bin（地形背景）
    rivers.tif                 → assets/rivers.png（河流叠加）
    06_pops_totals.txt ──────────→ assets/pops.tsv（人口）
  .eu5 存档 ──→ parse_save ────────→ assets/ownership.tsv（省份所有权）
                                    assets/vassals.tsv（宗藩关系）
                                    assets/merchandize.tsv（国家商品产出）
                                    assets/country_colors.tsv（国家颜色）
  EU5toGIS localization/ ──────────→ assets/province_names.tsv
                                    assets/country_names.tsv

启动时（服务端）：
  map.bin + *.tsv
    └──→ generate_world()
          ├── 所有权：ownership.tsv → 省份 owner 字段
          ├── 宗藩：vassals.tsv → GameState.vassals
          ├── 商品产出：merchandize.tsv → Country.produced_goods
          ├── 人口：pops.tsv → Province.pops
          ├── 名称：*_names.tsv → 显示名称
          └── 建筑：初始化农场与木炭窑
    └──→ 持久化至 RocksDB（后续启动直接加载）

启动时（客户端）：
  map.bin ──────────────────→ 合并省份网格（单 Mesh2d）
  terrain.bin ──────────────→ 合并地形网格（单 Mesh2d）
  rivers.png ───────────────→ 精灵叠加（NearestNeighbor 采样）
  country_colors.tsv ───────→ CountryColors 资源（标签 → RGBA）
  fonts/ ───────────────────→ CjkFont 资源

运行时（每 tick，仅 Playing 状态）：
  客户端 ──(Tick JSON)──→ 服务端
  服务端: advance() → 日期推进 → [每100tick: 生产→消费→增长]
  服务端 ──(StateSnapshot bincode)──→ 客户端
  客户端: 更新顶点颜色 → 更新 HUD → 更新首都标记
```

## 编译配置

### `.cargo/config.toml`

```toml
[build]
rustflags = ["-C", "link-arg=-fuse-ld=mold"]

[env]
CC = "clang"
CXX = "clang++"
```

- **clang**：编译 RocksDB C++ 依赖
- **mold**：超高速链接器，缩短增量编译时间
