# 数据提取指南

本文档描述从 EU5（Europa Universalis V）原始数据生成 Daboyi 地图编辑器所需资产的完整流程。所有步骤均可离线、可复现地执行。

---

## 目录结构约定

```
~/Playground/github/
├── EU5toGIS/              # GIS 数据仓库（EU5 地理数据集）
│   ├── datasets/
│   │   ├── locations.gpkg              # 省份多边形矢量数据（主数据源）
│   │   ├── ports.gpkg                  # 港口位置与海区连接
│   │   └── ...
│   └── localization/
│       └── simp_chinese/

│           └── location_names/         # 省份名称本地化（62个文件）
│               ├── location_names_l_simp_chinese.yml
│               ├── location_names_mandarin_chinese_l_simp_chinese.yml
│               └── ...

    ├── tools/
    │   ├── extract_translations.py     # 国家/省份名称提取

        ├── merchandize.tsv     ← 由 cargo run -p parse_save 生成
        └── country_colors.tsv  ← 由 cargo run -p parse_save 生成
```

## 第一步：获取 EU5toGIS 数据集

EU5toGIS 是由社区制作的 EU5 地理数据集，包含省份多边形矢量数据。

- 作者：DavidRMilos
- 下载地址：[Paradox 论坛帖子](https://forum.paradoxplaza.com/forum/threads/georeferenced-eu5-dataset-for-map-modding-via-gis.1903895/)

将仓库克隆至 `../EU5toGIS`（与 daboyi 同级目录）。

---

## 第二步：从 EU5 安装目录复制本地化文件

以下文件需从 EU5 Steam 安装目录（`<EU5_ROOT>`）手动复制到 EU5toGIS，因为这些文件不包含在 EU5toGIS 数据集中。

**Linux 默认路径：**

```
<EU5_ROOT> = ~/.local/share/Steam/steamapps/common/Europa Universalis V/game
```

### 2a. 国家名称本地化

```bash
cp "<EU5_ROOT>/main_menu/localization/simp_chinese/country_names_l_simp_chinese.yml" \
   ../EU5toGIS/localization/simp_chinese/
```

### 2b. 省份名称本地化（整个目录，62个文件）

```bash
mkdir -p ../EU5toGIS/localization/simp_chinese/location_names
cp -r "<EU5_ROOT>/main_menu/localization/simp_chinese/location_names/." \
   ../EU5toGIS/localization/simp_chinese/location_names/
```

**一键脚本（在 daboyi 目录下运行）：**

```bash
EU5="$HOME/.local/share/Steam/steamapps/common/Europa Universalis V/game"
EU5GIS="../EU5toGIS"
mkdir -p "$EU5GIS/localization/simp_chinese/location_names"
cp "$EU5/main_menu/localization/simp_chinese/country_names_l_simp_chinese.yml" \
   "$EU5GIS/localization/simp_chinese/"
cp -r "$EU5/main_menu/localization/simp_chinese/location_names/." \
   "$EU5GIS/localization/simp_chinese/location_names/"
```

## 第三步：生成翻译文件

从 EU5toGIS 的本地化文件提取国家和省份的中文名称：

```bash
uv run --project tools python tools/extract_translations.py
```

**输出：**
- `assets/country_names.tsv`：约 2,391 个国家标签 → 中文名称（简体）
- `assets/province_names.tsv`：约 29,492 个省份标签 → 中文名称

**加载优先级（省份名称）：**

1. 语言特定文件（最低，如 arabic、armenian 等）
2. 主 `location_names_l_simp_chinese.yml`
3. `location_names_mandarin_chinese_l_simp_chinese.yml`（最高，适用于中国/西藏地区）

---

## 第四步：编译地图二进制

从 `EU5toGIS/datasets/locations.gpkg` 和 `ports.gpkg` 生成地图数据：

```bash
cargo run --release -p mapgen
```

**输出：**
- `assets/map.bin`（~80 MB，约 22,688 个省份多边形，已被 `.gitignore` 排除）
- `assets/terrain.bin`（~40 MB，荒地/水域多边形）

首次运行编辑器时，还会基于 `map.bin` 自动生成：
- `assets/province_adjacency.bin`（省份相邻关系与边界折线缓存，用于加速后续启动）

> **注意：** 建议使用 `--release` 编译，Debug 模式处理大规模 GIS 数据速度很慢。
> `map.bin` 超过 GitHub 100 MB 限制，每次克隆仓库后需重新运行此步骤。

**处理流程：**

1. 读取 `locations.gpkg`，提取所有省份多边形（包括 ID、颜色、中心点）
2. 读取 `ports.gpkg`，补充海区连接信息
3. 对每个省份多边形进行三角剖分（earcut 算法）
4. 将经纬度转换为统一坐标系
5. 分离可游玩省份与地形/水域，分别写入 `map.bin` 和 `terrain.bin`

## 第五步：从 EU5 存档提取游戏数据

此步骤需要一个 EU5 **文本格式**存档（以 `SAV` 开头，非 ZIP 格式）。

```bash
cargo run -p parse_save -- /path/to/your.eu5 \
    assets/ownership.tsv \
    assets/vassals.tsv \
    assets/merchandize.tsv \
    assets/country_colors.tsv
```

**从存档读取的数据：**
- `compatibility.locations` — 省份名称列表（用于建立 ID 映射）
- `locations.locations` — 省份→所有者映射
- `dependency` — 宗主-附庸关系
- 各国 `last_month_produced` — 商品产出
- 各国 `color=rgb` — 国家颜色

**输出文件：**
- `assets/ownership.tsv` — 省份标签 → 所有者国家标签（编辑器默认着色）
- `assets/vassals.tsv` — 附庸标签 → 宗主标签
- `assets/merchandize.tsv` — 国家标签 + 商品名称 + 产量（降序）
- `assets/country_colors.tsv` — 国家标签 → RGB 颜色（0–255，用于政治地图初始颜色）

**在编辑器中的用途：**
- `ownership.tsv` 和 `country_colors.tsv` 作为编辑器打开时的默认着色状态（在 `coloring.json` 不存在时使用）
- `vassals.tsv` 和 `merchandize.tsv` 供省份信息面板展示参考信息

## 第六步：提取河流拓扑数据

河流使用单独的矢量提取脚本，从 EU5 的 `rivers.png` 生成 `assets/rivers.bin`：

```bash
uv run --project tools python tools/extract_rivers_vector.py
```

**输出：**
- `assets/rivers.bin`

该文件包含：

- 河流图节点（`nodes`）
- 带 `start_node` / `end_node` 的河段边（`edges`）
- 每条河段的控制点序列与宽度等级

提取脚本会先把骨架图中**连成一团的特殊像素（度数不等于 2）**折叠成单一逻辑节点，再把每条河段边的端点规范化到这些节点上。

这样客户端渲染河流分叉/汇流时可以直接使用显式拓扑与节点中心位置，生成“节点局部 mesh + 河段中段 strip”的连续几何，而不是在运行时从折线端点反推节点关系。

## 完整重建流程（新机器）

```bash
# 1. 克隆仓库
git clone https://github.com/xiaoshihou514/daboyi
cd daboyi

# 2. 复制 EU5 本地化文件到 EU5toGIS（见第二步）

# 3. 初始化 uv 管理的 Python 工具环境
uv sync --project tools

# 4. 生成翻译 TSV
uv run --project tools python tools/extract_translations.py

# 5. 编译地图二进制
cargo run --release -p mapgen

# 6. 从 EU5 存档提取数据
cargo run -p parse_save -- /path/to/your.eu5 \
    assets/ownership.tsv assets/vassals.tsv \
    assets/merchandize.tsv assets/country_colors.tsv

# 7. 提取河流拓扑
uv run --project tools python tools/extract_rivers_vector.py

# 8. 启动编辑器
cargo run -p client
```

第一次启动 `client` 时会生成 `assets/province_adjacency.bin`；后续只要 `map.bin` 未变化，编辑器就会直接复用该缓存。
