# 生产系统

## 概述

生产系统模拟省份内建筑的经济活动。每座建筑消耗原料和劳动力，产出商品。建筑的实际效率取决于劳动力供给和原料供给中的短板，形成维多利亚式的供应链瓶颈机制。

## 商品体系

游戏定义了 7 种商品（`Good`）：

| 商品 | 英文标识 | 类型 |
|------|----------|------|
| 粮食 | Grain | 基础消费品 |
| 衣物 | Clothing | 基础消费品 |
| 燃料 | Fuel | 基础消费品 |
| 工具 | Tools | 中间品/消费品 |
| 奢侈品 | Luxuries | 高级消费品 |
| 金属 | Metal | 中间品 |
| 建筑材料 | BuildingMaterials | 中间品 |

商品库存以 `GoodsBundle`（即 `HashMap<Good, f32>`）表示，存储在每个省份的 `stockpile` 字段中。

## 建筑体系

### 建筑类型定义

```rust

pub struct BuildingType {
    pub id: String,              // 唯一标识符
    pub name: String,            // 显示名称

    pub workers_per_level: u32,  // 每级所需工人数

```

## 生产机制

生产逻辑在 `server/src/game/production.rs` 中实现。

### 核心算法

```
对于每个省份的每座建筑：
1. 查找对应的 BuildingType 定义
2. 计算劳动力比率：
   worker_ratio = min(该阶级可用工人 / 所需工人总数, 1.0)
   所需工人总数 = workers_per_level × building.level
3. 计算原料比率（对每种原料）：
   input_ratio = min(库存量 / 所需量, 1.0)
   所需量 = 每级消耗 × building.level
4. 实际效率 = min(worker_ratio, 所有 input_ratio 的最小值)
5. 消耗原料 = 所需量 × 效率
6. 产出商品 = 每级产出 × building.level × 效率
```

### 效率瓶颈示例

假设一个 2 级纺织作坊（需要 400 小资产阶级工人，消耗 1.0 粮食/tick，产出 4.0 衣物/tick）：

| 场景 | 可用工人 | 库存粮食 | 劳动力比率 | 原料比率 | 实际效率 | 产出衣物 |
|------|---------|---------|-----------|---------|---------|---------|
| 满负荷 | 400+ | 1.0+ | 1.0 | 1.0 | 1.0 | 4.0 |
| 缺工人 | 200 | 1.0+ | 0.5 | 1.0 | 0.5 | 2.0 |
| 缺原料 | 400+ | 0.3 | 1.0 | 0.3 | 0.3 | 1.2 |
| 双重短缺 | 200 | 0.3 | 0.5 | 0.3 | 0.3 | 1.2 |

**关键设计**：效率取决于劳动力和所有原料中的最短板（min 函数），与维多利亚系列的产能瓶颈机制一致。

## 执行顺序

在每个 tick 中，`GameSimulation::advance()` 依次执行：

```
1. tick += 1
2. advance_date()          // 日期推进
3. produce_all()           // 所有省份生产
4. consume_and_grow_all()  // 所有省份人口消费与增长
```

生产在消费之前执行，确保本 tick 产出的商品可供本 tick 消费。

### 性能优化

`advance()` 中通过 Rust 字段分离借用避免克隆 `building_types`：

```rust
// Split borrow: provinces (mut) and building_types (shared) are disjoint fields.
let (provinces, building_types) = (&mut self.provinces, &self.building_types);
production::produce_all(provinces, building_types);
```

这利用了 Rust 借用检查器对同一结构体**不同字段**的独立借用能力，无需 `unsafe` 代码，同时避免了约 2 万省份 × 8 建筑类型的深拷贝。

## 世界生成中的初始建筑

每个省份初始配置两座建筑：

| 建筑 | 等级 | 理由 |
|------|------|------|
| 农场 (farm) | 1 | 确保基本粮食供给 |
| 木炭窑 (charcoal_kiln) | 1 | 确保基本燃料供给 |

初始库存：粮食 20、衣物 5、燃料 3。这为经济启动提供了缓冲。

## 与其他系统的关联

- **人口系统**：提供劳动力（`worker_class` 对应的 Pop 群体 `size`）；消费生产产出。
- **省份系统**：建筑和库存归属于省份；生产仅使用本省资源。
- **地图系统**：生产地图模式（MapMode::Production）以总库存值为热力图指标。

## 当前局限与规划

- **无省际贸易**：库存完全隔离于本省，缺乏商品流通机制。
- **无建造指令**：`IssueOrder` 框架已搭建但建筑建造尚未实现。
- **无市场定价**：商品无价格，消费按固定比率执行。
- **无科技影响**：建筑效率不受科技水平影响。
