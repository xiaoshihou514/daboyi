# 地图系统

## 概述

地图系统负责将 EU5toGIS 游戏设计的省份数据转换为游戏可用的多边形，并在客户端高效渲染。整个流程分为两阶段：离线地图编译（mapgen 工具）和运行时地图渲染（客户端 MapPlugin）。

### 几何格式

- **存储格式**：GeoPackage Binary (GPB) = GP头(8字节) + 包络线(32字节,xy) + WKB MultiPolygon
- **投影**：Gall 立体投影（自定义 SRS 100000）
- **逆变换**：`lon = x/(R·cos45°)`, `lat = 2·atan(y/(R·(1+cos45°)))`, R = 6378137

### 可玩省份过滤规则

排除以下 topography 值的要素：
`coastal_ocean`、`ocean`、`inland_sea`、`deep_ocean`、`narrows`、`lakes`、`high_lakes`、`atoll`、`salt_pans`，以及任何包含 `wasteland` 的值。

## 离线地图编译（mapgen）

### 工具位置

`tools/mapgen/src/main.rs`

### 依赖

- `rusqlite`：读取 GPKG 文件（SQLite 数据库）
- `geo`：多边形简化（Ramer-Douglas-Peucker 算法）
- `earcutr`：耳切三角剖分

### 处理流程

```
ports.gpkg ──→ 读取 ports 图层 ──→ 构建 tag→SeaZone 映射
                                        │
locations.gpkg ──→ SQL 查询 ──→ GPB/WKB 解析 ──→ Gall→WGS84 坐标变换
                                                     │
                                        可玩省份过滤（21,111/28,573）
                                                     │
                                        多边形简化（epsilon=0.005）
                                                     │
                                        耳切三角剖分
                                                     │
                                        关联港口海域数据
                                                     │
                                                 map.bin (~80MB)
```

### 核心函数

| 函数 | 功能 |
|------|------|
| `gall_stereo_to_wgs84(x, y)` | Gall 立体投影逆变换为经纬度 |
| `parse_gpb_geometry(blob)` | 解析 GPB 头 + WKB MultiPolygon |
| `parse_wkb_multipolygon(data, le)` | 递归解析 WKB 多边形结构 |
| `process_polygons(rings)` | 简化 + 三角剖分 + 质心计算 |
| `is_non_playable(topo)` | 判断地形是否不可玩 |

```rust
// shared/src/map.rs

pub struct MapData {
    pub provinces: Vec<MapProvince>,

}
```

## 地形背景渲染（TerrainPlugin）

### 位置

`client/src/terrain.rs`

### 职责

TerrainPlugin 渲染所有不可玩区域（海洋、湖泊、荒原等）作为地图背景，并叠加河流图像。

### 地形多边形渲染

所有不可玩地形多边形合并为**单个 Bevy Mesh2d 实体**，顶点颜色由 mapgen 烘焙。渲染时使用 `ColorMaterial::default()`（白色），与顶点颜色相乘得到最终颜色。

渲染层级：Z = -1.0（在可玩省份网格 Z=0.0 之下）。

共渲染 3 份横向平铺（-360°、0°、+360°），支持地图横向无缝滚动。

|----------|------------------|----------------|

| `ocean` | [0.039, 0.165, 0.416] | #0A2A6A 海洋蓝 |
| `coastal_ocean` | [0.051, 0.227, 0.604] | #0D3A9A 近海蓝 |

### 河流叠加精灵

`assets/rivers.png`（4096×2048 RGBA）作为精灵叠加在地形层之上：

- 覆盖 360°×180° 世界范围（与地图坐标系对齐）
- Z = 0.5（位于省份网格 Z=0.0 之上）
- `flip_y: true`：地理 PNG 行 0 对应北纬 90°，即 Bevy 世界 Y 轴最大值，需翻转以正确对齐
- `ImageSampler::nearest()`：使用最近邻采样保持细线清晰，避免线性插值导致细线消失为断点

### 窗口背景色

`ClearColor` 设置为海洋蓝 `Color::srgb(0.1, 0.26, 0.55)`，使地图边缘和数据缺失区域不显示黑色背景。


## 运行时地图渲染（MapPlugin）

### 位置

`client/src/map.rs`

### 合并网格渲染

所有 ~21,111 个可玩省份的三角形合并为**单个 Bevy Mesh2d 实体**，使用顶点颜色区分省份颜色。渲染层级 Z = 0.0。

### 地图着色模式

| 模式 | 快捷键 | 着色逻辑 |
|------|--------|----------|
| 省份 (Province) | 1 | EU5 原始识别颜色（烘焙于 map.bin） |
| 人口 (Population) | 2 | 热力图：总人口 / 全局最大人口 |
| 生产 (Production) | 3 | 热力图：总库存 / 全局最大库存 |
| 地形 (Terrain) | 4 | 按省份 topography 字段固定颜色 |
| 政治 (Political) | 5 | 有主省份：FNV 哈希生成确定性国家颜色（范围 [0.2, 0.75]）；无主省份：中性灰 [0.55, 0.55, 0.55] |



### 智能变更检测

- 经济系统每 100 tick 运行一次，因此仅在经济 tick 边界进行完整重着色
- 选中省份变更时，仅更新涉及的 ~60 个顶点（而非全部数百万个）
- 地图模式切换时强制完整重着色

### 省份信息面板

点击省份后在右侧面板显示：
- 省份名称与所属国家
- 地形、植被、气候
- 资源类型（如有）
- 港口适宜度与连接海域（如有）
- 人口分布与需求满足度
- 建筑列表与等级
- 库存商品

### 相机控制

| 操作 | 输入 |
|------|------|
| 平移 | 鼠标右键拖拽 |
| 缩放 | 滚轮 |

### 省份点击检测
