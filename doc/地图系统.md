# 地图系统

本文档描述 Daboyi 地图编辑器中省份地图的渲染架构，包括省份网格、边界、地形背景、首都标记及地图模式切换。

---

## 整体结构

地图渲染由以下四个插件协作完成：

| 插件 | 职责 |
|------|------|
| `MapPlugin` | 省份多边形渲染、着色逻辑、地图模式切换 |
| `BordersPlugin` | 相邻省份边界线渲染 |
| `TerrainPlugin` | 荒地与水域背景多边形 |
| `CapitalsPlugin` | 首都星形标记 |

---

## 省份网格渲染（MapPlugin）

### 数据来源

省份几何数据来自 `assets/map.bin`（bincode 格式），由 `mapgen` 工具从 EU5toGIS `locations.gpkg` 生成。每个 `MapProvince` 包含：

- `id: u32` — 省份唯一标识符
- `color: [u8; 3]` — EU5 原始识别颜色（省份模式用）
- `triangles: Vec<[f32; 2]>` — 三角剖分后的顶点坐标（经纬度）
- `center: [f32; 2]` — 省份中心点（用于首都定位）

### 合并网格策略

启动时将所有省份多边形**合并为单一 `Mesh2d`**，以减少 draw call：

1. 遍历所有 `MapProvince`，将顶点追加到统一顶点缓冲区
2. 记录每个省份的顶点起始偏移量（存储在 `ProvinceVertexOffset` 资源中）
3. 着色时，仅更新对应省份顶点段的颜色属性（`ATTRIBUTE_COLOR`），不重建网格

这种方式在着色模式切换或涂色操作时，只需更新 CPU 端颜色缓冲区并通知 GPU 刷新，性能远优于每省份独立 `Entity`。

### 省份着色逻辑

着色由 `map/mod.rs` 中的系统负责，根据当前 `MapMode` 和编辑器资源计算每个省份的颜色：

**省份模式（`MapMode::Province`）：**
- 使用 EU5 原始识别颜色（`MapProvince.color`）
- 荒地/水域省份显示深灰色

**地形模式（`MapMode::Terrain`）：**
- 荒地显示深褐色，水域显示深蓝色
- 可玩省份显示统一浅绿色

**政治模式（`MapMode::Political`，默认）：**
- 查询 `AdminAssignments`：若省份有行政区指定，使用行政区颜色（向上继承至国家）
- 查询 `MapColoring`：若省份有国家指定，使用国家颜色
- 无主省份显示中性灰 `[0.55, 0.55, 0.55, 1.0]`

## 边界渲染（BordersPlugin）

`borders.rs` 在相邻省份之间渲染边界线。

### 边界类型

- **国家边界**：相邻省份属于不同国家时渲染（较粗，深色）
- **行政区边界**：相邻省份属于同一国家但不同行政区时渲染（较细）
- **未归属边界**：不渲染（灰色区域之间无边界）

### 实现方式

边界线同样合并为单一网格，以线段列表的形式存储。当着色状态发生变化时，系统重新计算需要显示的边界并更新网格。

为了避免每次启动都重新扫描全部省份边界，`BordersPlugin` 会优先从 `assets/province_adjacency.bin` 读取预计算的相邻关系缓存：

- 缓存内容：相邻省份对 + 已链化/平滑后的边界折线
- 失效条件：缓存版本变化，或 `map.bin` 的省份数量与缓存记录不一致
- 缓存不存在时：客户端会在首次启动时重新计算，并把结果写回 `assets/province_adjacency.bin`

这样后续启动时通常只需反序列化缓存，不再重复进行 6 万多个相邻省份对的边界构建。

## 地形背景渲染（TerrainPlugin）

地形数据来自 `assets/terrain.bin`，包含荒地、山地、极地和水域多边形，用于填充不可游玩区域的背景。

- 荒地：棕褐色 `[0.45, 0.40, 0.35, 1.0]`
- 水域/海洋：深蓝色 `[0.18, 0.30, 0.50, 1.0]`

与省份网格类似，所有地形多边形合并为单一 `Mesh2d`，在省份层之下渲染（Z 轴更低）。

### 河流渲染

河流数据来自 `assets/rivers.bin`。该文件不再只保存“孤立折线”，而是显式保留河流图拓扑，并且在提取阶段把同一个汇流/分叉像素团块折叠为单一逻辑节点：

- `nodes`：河流图节点（汇流点、分叉点、端点、闭环锚点）
- `edges`：连接两个节点的河段，每条边包含：
  - 平滑前的控制点序列
  - 宽度等级
  - `start_node` / `end_node`

客户端渲染流程：

1. 对每条河段边做 Chaikin 平滑
2. 先计算整条河段的左右边界，但**不直接渲染首尾贴近节点的那一段**
3. 以**节点 id** 为准收集所有相邻河段在节点处的首/尾 quad 轮廓
4. 用这些 quad 轮廓合并出一个节点局部 mesh，由节点“拥有”接缝区域
5. 河段主体只渲染去掉首尾节点区域后的中间 strip

这样河流接缝不再依赖“浮点端点量化猜测”，也不再是“完整河段 + 额外补丁”的叠加关系，而是直接使用提取阶段保留下来的图结构生成一体化局部网格。

## 首都标记（CapitalsPlugin）

`capitals.rs` 在每个国家的首都省份中心渲染一个星形标记。

- 标记位置：`EditorCountry.capital_province` 对应的 `MapProvince.center`
- 标记颜色：与国家颜色对比的高亮色（白色或黑色描边）
- 当国家被删除或首都更改时，标记自动更新

## 相机控制与省份交互（interact.rs）

### 相机控制

| 操作 | 行为 |
|------|------|
| 中键拖拽 | 平移视角（`Camera2d` 位移） |
| 滚轮 | 缩放（调整 `OrthographicProjection.scale`） |

相机位移量根据当前缩放比例归一化，确保任何缩放级别下拖拽速度一致。

### 省份选择与画笔涂色

**点击选择：**

1. 将鼠标屏幕坐标转换为世界坐标
2. 用世界坐标查询 `ProvincePickMap`（空间哈希，省份 ID 快速查找）
3. 更新 `SelectedProvince` 资源，触发右侧信息面板刷新

**画笔涂色（左键拖拽）：**

1. 鼠标按下时开始记录拖拽轨迹
2. 每帧检测鼠标下方的省份
3. 若存在激活的国家（`ActiveCountry`）或行政区（`ActiveArea`），将省份写入对应资源（`MapColoring` 或 `AdminAssignments`）
4. 触发着色系统重新计算受影响省份的顶点颜色

## 地图模式切换

地图模式由 `MapMode` 枚举控制，通过键盘快捷键或工具栏按钮切换：

| 快捷键 | 模式 | 说明 |
|--------|------|------|
| `1` | 省份模式 | EU5 原始省份识别颜色 |
| `2` | 地形模式 | 荒地/水域高亮 |
| `3` | 政治模式 | 国家/行政区着色（默认） |

切换模式时，系统遍历所有省份顶点段，重新计算并写入颜色值。

---

## 坐标系

- 世界坐标使用 **经纬度**（WGS84）作为单位，X 轴为经度（-180°～180°），Y 轴为纬度（-90°～90°）
- `mapgen` 在生成 `map.bin` 时将 GPKG 投影坐标转换为经纬度
- `OrthographicProjection` 的初始缩放设置为覆盖全球范围

| `assets/province_names.tsv` | TSV | 省份标签 → 中文名称 |
