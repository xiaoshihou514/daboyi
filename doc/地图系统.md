# 地图系统

## 概述

地图系统负责将 EU5toGIS 游戏设计的省份数据转换为游戏可用的多边形，并在客户端高效渲染。整个流程分为两阶段：离线地图编译（mapgen 工具）和运行时地图渲染（客户端 MapPlugin）。

## 地理数据源

### EU5toGIS 数据集

### 几何格式

- **存储格式**：GeoPackage Binary (GPB) = GP头(8字节) + 包络线(32字节,xy) + WKB MultiPolygon
- **投影**：Gall 立体投影（自定义 SRS 100000）
- **逆变换**：`lon = x/(R·cos45°)`, `lat = 2·atan(y/(R·(1+cos45°)))`, R = 6378137

### 可玩省份过滤规则

排除以下 topography 值的要素：
`coastal_ocean`、`ocean`、`inland_sea`、`deep_ocean`、`narrows`、`lakes`、`high_lakes`、`atoll`、`salt_pans`，以及任何包含 `wasteland` 的值。

## 离线地图编译（mapgen）

### 工具位置

`tools/mapgen/src/main.rs`

### 依赖

- `rusqlite`：读取 GPKG 文件（SQLite 数据库）
- `geo`：多边形简化（Ramer-Douglas-Peucker 算法）
- `earcutr`：耳切三角剖分

### 处理流程

```
ports.gpkg ──→ 读取 ports 图层 ──→ 构建 tag→SeaZone 映射
                                        │
locations.gpkg ──→ SQL 查询 ──→ GPB/WKB 解析 ──→ Gall→WGS84 坐标变换
                                                     │
                                        可玩省份过滤（21,111/28,573）
                                                     │
                                        多边形简化（epsilon=0.005）
                                                     │
                                        耳切三角剖分
                                                     │
                                        关联港口海域数据
                                                     │
                                                 map.bin (~80MB)
```

### 核心函数

| 函数 | 功能 |
|------|------|
| `gall_stereo_to_wgs84(x, y)` | Gall 立体投影逆变换为经纬度 |
| `parse_gpb_geometry(blob)` | 解析 GPB 头 + WKB MultiPolygon |
| `parse_wkb_multipolygon(data, le)` | 递归解析 WKB 多边形结构 |
| `process_polygons(rings)` | 简化 + 三角剖分 + 质心计算 |
| `is_non_playable(topo)` | 判断地形是否不可玩 |

## 输出格式

```rust
// shared/src/map.rs

pub struct MapData {
    pub provinces: Vec<MapProvince>,
}

pub struct MapProvince {
    pub id: u32,                          // 省份索引
    pub tag: String,                      // EU5 省份标签
    pub name: String,                     // 地名（tag 的副本）
    pub topography: String,               // 地形类型
    pub vegetation: String,               // 植被类型
    pub climate: String,                  // 气候类型
    pub raw_material: String,             // 原始资源（可为空）
    pub harbor_suitability: f32,          // 港口适宜度 (0.0–1.0)
    pub port_sea_zone: Option<String>,    // 连接的海域名称
    pub boundary: Vec<Vec<[f32; 2]>>,     // 多边形环
    pub vertices: Vec<[f32; 2]>,          // 三角剖分后的顶点
    pub indices: Vec<u32>,                // 三角形索引
    pub centroid: [f32; 2],               // 质心坐标（经纬度）
}
```

## 运行时地图渲染（MapPlugin）

### 位置

`client/src/map.rs`

### 合并网格渲染

所有 ~21,000 个省份的三角形合并为**单个 Bevy Mesh2d 实体**，使用顶点颜色区分省份颜色。

### 地图着色模式

| 模式 | 快捷键 | 着色逻辑 |
|------|--------|----------|
| 政治 (Political) | 1 | 按 `province.owner` 国家标签哈希生成确定性颜色 |
| 人口 (Population) | 2 | 热力图：总人口 / 全局最大人口 |
| 生产 (Production) | 3 | 热力图：总库存 / 全局最大库存 |

### 智能变更检测

- 经济系统每 100 tick 运行一次，因此仅在经济 tick 边界进行完整重着色
- 选中省份变更时，仅更新涉及的 ~60 个顶点（而非全部数百万个）
- 地图模式切换时强制完整重着色

### 省份信息面板

点击省份后在右侧面板显示：
- 省份名称与所属国家
- 地形、植被、气候
- 资源类型（如有）
- 港口适宜度与连接海域（如有）
- 人口分布与需求满足度
- 建筑列表与等级
- 库存商品

### 相机控制

| 操作 | 输入 |
|------|------|
| 平移 | 鼠标右键拖拽 |
| 缩放 | 滚轮 |

### 省份点击检测

左键点击时：屏幕坐标 → 世界坐标 → 逆序遍历 → AABB 过滤 → 射线法测试 → 选中省份
