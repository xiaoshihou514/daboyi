# 地图系统

## 概述

地图系统负责将真实世界的地理数据转换为游戏可用的省份多边形，并在客户端高效渲染。整个流程分为两阶段：离线地图编译（mapgen 工具）和运行时地图渲染（客户端 MapPlugin）。

## 地理数据源

### 数据来源

游戏使用预打包的 GeoJSON 数据，存放在 `/home/xiaoshihou/Playground/shared/geojson/`：

| 文件 | 内容 | 行政级别 | 省份数量 |
|------|------|----------|---------|
| `cn_adm3.geojson` | 中国各区县 | ADM3（县级） | ~2,891 |
| `world_adm2.geojson` | 全球各区域（排除中国） | ADM2（地级） | ~46,979 |

**中国优先策略**：中国使用更高精度的 ADM3 数据（县级），全球其他地区使用 ADM2（地级）。全球数据中排除中国条目以避免重复。

### 数据模式

两种文件的 GeoJSON properties 字段格式不同：

**中国数据**（`cn_adm3.geojson`）：
```json
{
  "name": "鄂城区",
  "gb": "156420704"
}
```
- `name`: 中文地名
- `gb`: 国标行政区划代码
- 固定国家代码 `CHN`

**全球数据**（`world_adm2.geojson`）：
```json
{
  "shapeName": "Berlin",
  "shapeID": "DEU-ADM2-...",
  "shapeGroup": "DEU",
  "shapeType": "ADM2"
}
```
- `shapeName`: 地区名称
- `shapeID`: 唯一标识
- `shapeGroup`: ISO 3166 国家代码

## 离线地图编译（mapgen）

### 工具位置

`tools/mapgen/src/main.rs`

### 处理流程

```
cn_adm3.geojson ──┐
                   ├──→ 解析 GeoJSON ──→ 简化多边形 ──→ 三角剖分 ──→ map.bin
world_adm2.geojson ┘
```

### 详细步骤

对每个 GeoJSON Feature：

1. **元数据提取**（`extract_meta()`）：
   - 根据字段格式判断数据来源（CN 或 world）
   - 提取 name、gadm_id、country_code

2. **几何解析**：
   - 支持 `Polygon` 和 `MultiPolygon` 几何类型
   - 每个多边形提取为外环 + 孔洞环

3. **多边形简化**（`geo::Simplify`）：
   - epsilon = 0.001°（赤道约 100 米精度）
   - 将顶点数减少约 90%

4. **三角剖分**（`earcutr`）：
   - 使用耳切法（ear-clipping）将多边形分解为三角形
   - 处理外环和孔洞的正确拓扑关系

5. **质心计算**：
   - 外环顶点的算术平均值，用于标签放置

6. **坐标投影**：
   - 采用等距圆柱投影（equirectangular）：经度 → x，纬度 → y
   - 简单直接，适合全球尺度的 2D 可视化

### 输出格式

```rust
// shared/src/map.rs

pub struct MapData {
    pub provinces: Vec<MapProvince>,
}

pub struct MapProvince {
    pub id: u32,                          // 省份索引（= 在数组中的位置）
    pub gadm_id: String,                  // 原始数据 ID
    pub name: String,                     // 地名
    pub country_code: String,             // ISO 国家代码
    pub boundary: Vec<Vec<[f32; 2]>>,     // 多边形环（外环 + 孔洞）
    pub vertices: Vec<[f32; 2]>,          // 三角剖分后的顶点
    pub indices: Vec<u32>,                // 三角形索引
    pub centroid: [f32; 2],               // 质心坐标
}
```

- 使用 bincode 序列化存储为 `assets/map.bin`
- `boundary` 保留原始多边形用于点击检测
- `vertices` / `indices` 是预三角剖分的网格数据用于渲染

### 省份编号规则

- 中国省份在前，全球省份在后
- ID = 数组索引，在 MapData 和 GameState 中一一对应
- 中国约 2,891 个省份 → ID 0~2890
- 全球约 46,979 个省份 → ID 2891~49869

## 运行时地图渲染（MapPlugin）

### 位置

`client/src/map.rs`

### 核心资源

| 资源 | 类型 | 说明 |
|------|------|------|
| `MapResource` | Resource | 加载的 MapData，包含所有省份几何 |
| `SelectedProvince` | Resource | 当前选中的省份 ID |
| `MapMode` | Resource | 地图着色模式 |
| `ProvinceVertexMap` | Resource | 省份 ID → 顶点范围的映射 |
| `LastColorState` | Resource | 上次着色时的状态快照（变更检测用） |

### 合并网格渲染

这是最关键的性能优化。所有 ~50,000 个省份的三角形合并为**单个 Bevy Mesh2d 实体**，使用顶点颜色（`ATTRIBUTE_COLOR`）区分省份颜色。

```
传统方案（旧版）：         优化方案（当前）：
50,000 个 Entity          1 个 Entity
50,000 个 Mesh            1 个合并 Mesh
50,000 次 Draw Call       1 次 Draw Call
```

#### 合并流程

```rust
// load_map() 启动系统

对每个省份 mp：
  base_idx = 当前总顶点数
  z = if mp.country_code == "CHN" { 0.1 } else { 0.0 }

  对每个顶点 v：
    positions.push([v.x, v.y, z])
    colors.push(country_color_rgba(mp.country_code))

  对每个三角形索引 idx：
    indices.push(idx + base_idx)

  ranges.push((start, vertex_count))
```

`ProvinceVertexMap.ranges` 记录每个省份在全局顶点数组中的起始位置和顶点数量，用于后续的颜色更新。

### Z 轴分层（Z-fighting 修复）

中国和全球数据对于争议领土（台湾、藏南等）存在重叠多边形。通过 Z 轴分层解决闪烁问题：

| 数据源 | Z 值 | 渲染优先级 |
|--------|------|-----------|
| 中国省份 | 0.1 | 高（覆盖在上层） |
| 全球省份 | 0.0 | 低（被中国省份覆盖） |

### 地图着色模式

通过数字键 1/2/3 切换，颜色通过修改合并网格的顶点颜色实现：

| 模式 | 快捷键 | 着色逻辑 |
|------|--------|----------|
| 政治 (Political) | 1 | 按 `province.owner` 国家代码哈希生成确定性颜色 |
| 人口 (Population) | 2 | 热力图：总人口 / 全局最大人口，蓝→绿→红 |
| 生产 (Production) | 3 | 热力图：总库存 / 全局最大库存，蓝→绿→红 |

#### 热力图颜色映射

```
t = 0.0  →  蓝色 (冷/低)
t = 0.5  →  绿色 (中等)
t = 1.0  →  红色 (热/高)
```

选中的省份会在基础颜色上增加 +0.25 亮度。

### 变更检测优化

`color_provinces()` 系统在每帧检查是否需要重新着色：

```rust
if last.tick == gs.tick && last.mode == Some(*mode) && last.selected == selected.0 {
    return;  // 跳过——无变更
}
```

仅当以下条件之一成立时才重建颜色缓冲：
- 游戏 tick 推进（收到新状态快照）
- 地图模式切换
- 选中省份变更

这将每帧 ~50K 顶点的颜色写入降低为仅在状态变更时执行。

### 相机控制

| 操作 | 输入 | 说明 |
|------|------|------|
| 平移 | 鼠标右键拖拽 | 按投影缩放比例移动相机 |
| 缩放 | 滚轮 | 调整 `OrthographicProjection.scale`（范围 0.01–10.0） |

### 省份点击检测

左键点击时执行省份识别：

1. **屏幕坐标 → 世界坐标**：通过 Bevy 的 `viewport_to_world_2d` 转换
2. **逆序遍历**：从后向前遍历省份列表（中国省份在后，优先检测）
3. **包围盒预过滤**：计算外环的 AABB，快速排除不可能命中的省份
4. **射线法点位多边形测试**：对通过包围盒的省份执行精确的点-多边形测试
5. **孔洞排除**：如果点在外环内但同时在孔洞内，则不算命中

```
点击流程：
屏幕坐标 → 世界坐标 → 逆序遍历 → AABB 过滤 → 射线法测试 → 选中省份
```

## 省份与游戏状态的对应

省份 ID 在地图数据和游戏状态中保持一一对应：

```
MapData.provinces[i].id == i == GameState.provinces[i].id
```

这使得根据 `SelectedProvince` 的 ID 可以同时在 `MapResource` 中查找地理信息，在 `LatestGameState` 中查找游戏数据。
