# 地图系统

## 概述

地图系统负责将真实世界的地理数据转换为游戏可用的省份多边形，并在客户端高效渲染。整个流程分为两阶段：离线地图编译（mapgen 工具）和运行时地图渲染（客户端 MapPlugin）。

## 地理数据源

### 数据来源

游戏使用预打包的 GeoJSON 数据，存放在 `/home/xiaoshihou/Playground/shared/geojson/`：

| 文件 | 内容 | 行政级别 | 原始省份数量 |
|------|------|----------|-------------|
| `cn_adm3.geojson` | 中国各区县 | ADM3（县级） | ~2,891 |
| `world_adm2.geojson` | 全球各区域（排除中国、台湾、争议领土） | ADM2（地级） | ~46,609 |

**中国优先策略**：中国使用更高精度的 ADM3 数据（县级），全球其他地区使用 ADM2（地级）。mapgen 编译时排除以下数据以消除领土重叠：
- `CHN`：中国条目（已使用 cn_adm3 覆盖）
- `TWN`：台湾条目（中国 ADM3 数据已包含台湾各区县）
- 印度争议区：Tawang（与中国错那/隆子重叠）、Leh(Ladakh)（含阿克赛钦）

## 省份细分系统

### 问题

原始 ADM2 数据中存在大量面积过大的省份（~1,750 个 > 1 deg² ≈ 12,300 km²），最大的如南极洲 1,712 deg²、努纳武特 166 deg²。这些巨型省份不利于游戏玩法的颗粒度。

### 解决方案

在 `tools/mapgen/src/subdivide.rs` 中实现网格细分：

```
大省份 → 计算包围盒 → 确定网格维度 → 逐格裁剪多边形 → 生成子省份
```

#### 细分算法

1. **面积计算**：使用鞋带公式计算多边形面积（度²）
2. **阈值判断**：面积 > 1.0 deg² 的省份需要细分
3. **网格维度**：`n_cells = ceil(area / threshold)`，按宽高比分配行列数
4. **多边形裁剪**：使用 geo crate 的 `BooleanOps::intersection()` 将多边形与网格单元求交
5. **重三角剖分**：对每个裁剪后的子多边形重新进行耳切三角剖分
6. **命名规则**：子省份命名为 `"{原名} {行}-{列}"`

#### 细分效果

| 指标 | 细分前 | 细分后 |
|------|--------|--------|
| 总省份数 | ~49,500 | ~61,308 |
| 被细分省份 | 0 | ~2,027 |
| 最大省份面积 | ~1,712 deg² | ≤ 1.0 deg² |

## 离线地图编译（mapgen）

### 工具位置

- `tools/mapgen/src/main.rs` — 主入口
- `tools/mapgen/src/subdivide.rs` — 省份细分逻辑

### 处理流程

```
cn_adm3.geojson ──┐
                   ├──→ 解析 GeoJSON ──→ 简化多边形 ──→ 三角剖分 ──→ 细分大省份 ──→ map.bin
world_adm2.geojson ┘
```

### 输出格式

```rust
// shared/src/map.rs

pub struct MapData {
    pub provinces: Vec<MapProvince>,
}

pub struct MapProvince {
    pub id: u32,                          // 省份索引
    pub gadm_id: String,                  // 原始数据 ID
    pub name: String,                     // 地名
    pub country_code: String,             // ISO 国家代码
    pub boundary: Vec<Vec<[f32; 2]>>,     // 多边形环
    pub vertices: Vec<[f32; 2]>,          // 三角剖分后的顶点
    pub indices: Vec<u32>,                // 三角形索引
    pub centroid: [f32; 2],               // 质心坐标
}
```

## 运行时地图渲染（MapPlugin）

### 位置

`client/src/map.rs`

### 合并网格渲染

所有 ~61,000 个省份的三角形合并为**单个 Bevy Mesh2d 实体**，使用顶点颜色区分省份颜色。

### 地图着色模式

| 模式 | 快捷键 | 着色逻辑 |
|------|--------|----------|
| 政治 (Political) | 1 | 按 `province.owner` 国家标签哈希生成确定性颜色 |
| 人口 (Population) | 2 | 热力图：总人口 / 全局最大人口 |
| 生产 (Production) | 3 | 热力图：总库存 / 全局最大库存 |

### 变更检测优化

仅当游戏 tick 推进、地图模式切换或选中省份变更时才重建颜色缓冲。

### 相机控制

| 操作 | 输入 |
|------|------|
| 平移 | 鼠标右键拖拽 |
| 缩放 | 滚轮 |

### 省份点击检测

左键点击时：屏幕坐标 → 世界坐标 → 逆序遍历 → AABB 过滤 → 射线法测试 → 选中省份

## 省份与游戏状态的对应

```
MapData.provinces[i].id == i == GameState.provinces[i].id
```
